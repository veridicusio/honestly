# Additional License Terms and Contribution Requirements

## Attribution for All Contributions and Derivatives

This document specifies additional requirements for contributions to and derivatives of the "honestly" repository.

### Mandatory Contribution Attribution

All contributions, modifications, forks, and derivative works of this software MUST display the following attribution:

```
Contributions belong to original author: aresforblue-ai
Original Repository: https://github.com/aresforblue-ai/honestly
```

### Where Attribution Must Appear

This attribution statement MUST be included in:

1. **Source Code Files**
   - In file headers or top-level comments of any modified or new files
   - Example for JavaScript/TypeScript:
     ```javascript
     /**
      * Contributions belong to original author: aresforblue-ai
      * Original Repository: https://github.com/aresforblue-ai/honestly
      */
     ```
   - Example for Python:
     ```python
     # Contributions belong to original author: aresforblue-ai
     # Original Repository: https://github.com/aresforblue-ai/honestly
     ```

2. **Documentation Files**
   - README.md files in any fork or derivative
   - API documentation
   - User guides and tutorials
   - Developer documentation

3. **User Interfaces**
   - About pages in web applications
   - Help sections in CLI tools
   - Footer sections in web interfaces
   - Credits screens in desktop/mobile applications

4. **Build Artifacts**
   - Compiled binaries metadata
   - Container image labels
   - Package manifests (package.json, setup.py, etc.)

5. **Distribution Channels**
   - npm package descriptions
   - PyPI project descriptions
   - Docker Hub descriptions
   - GitHub/GitLab repository descriptions

### Contribution Guidelines

#### For Pull Requests to This Repository

When contributing to the original repository:

1. By submitting a pull request, you agree that your contributions are subject to the AGPL-3.0-only license
2. You acknowledge that the original author (aresforblue-ai) retains copyright to the project
3. You grant the project maintainers the right to use, modify, and distribute your contributions
4. Your contributions must not violate any third-party copyrights or licenses

#### For Forks and Derivatives

When creating forks or derivative works:

1. **Retain All Original Attribution**
   - Do not remove or obscure the original author attribution
   - Keep all copyright notices intact
   - Maintain links to the original repository

2. **Add Your Own Attribution**
   - You may add your own copyright notices for your modifications
   - Your notices must not claim ownership of the original work
   - Format: `Modifications Copyright (C) [Year] [Your Name]`

3. **Document Changes**
   - Maintain a CHANGELOG or similar file documenting your modifications
   - Clearly indicate which parts are original and which are modified
   - Include dates of modifications

4. **Comply with AGPL-3.0**
   - Release your modifications under AGPL-3.0-only
   - Make source code available for any network-accessible deployments
   - Follow all requirements in the LICENSE file

### On-Chain Proof Requirements for Derivatives

If you deploy a derivative work:

1. **Record Your Deployment**
   - Follow the on-chain proof requirements in the LICENSE file
   - Include a reference to your fork/derivative in the proof
   - Link back to the original repository

2. **Notify the Original Repository**
   - Submit an issue or PR documenting your derivative
   - Include the blockchain transaction/proof link
   - Describe your modifications briefly
