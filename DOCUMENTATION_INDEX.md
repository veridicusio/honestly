## üîß Development Workflow

### Setup & Configuration
```bash
# Quick start
make install    # Install all dependencies
make up         # Start development environment
make test       # Run all tests
make down       # Stop services
```

### Common Tasks
```bash
# Backend development
cd backend-python && uvicorn api.app:app --reload

# ConductMe development
cd conductme && npm run dev

# ZKP circuit compilation
cd backend-python/zkp && npm run build:all

# Run tests
pytest tests/ -v --cov
npm test
```

### Pre-commit Hooks
- Black (Python formatting)
- Ruff (Python linting)
- Prettier (JS/TS formatting)
- ESLint (JS/TS linting)

## üéØ Documentation Quality Standards

All documentation in this repository follows these standards:

1. **Clear Structure** ‚Äî Consistent heading hierarchy, TOC for long docs
2. **Code Examples** ‚Äî All examples are tested and working
3. **Cross-References** ‚Äî Links to related documentation
4. **Maintenance** ‚Äî Last updated date on all major docs
5. **Audience-Aware** ‚Äî Clear indication of target audience
6. **Professional Formatting** ‚Äî Proper markdown, tables, diagrams
7. **Visual Elements** ‚Äî Badges, emojis, diagrams where appropriate
8. **Actionable** ‚Äî Step-by-step instructions, not just theory
